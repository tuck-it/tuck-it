{% load web_extras %} {% comment %} Quick capture — the product's most-opened dialog (C, the sidebar button, every page header). It is shown/hidden by Alpine rather than swapped in by htmx, so it needs its own dialog semantics and focus handling: role/aria-modal so it is announced as a dialog at all, a Tab trap so focus cannot land on a control behind the scrim, and dialogSync to hand focus back to whatever opened it. This dialog creates a TICKET, always. Area is filing ("which part of the product"), not commitment ("are we doing this") — two different axes. It used to include _slice_fields.html, where the Area select doubled as a Ticket/Slice fork: picking an area produced a `planned` Slice, so there was no way to park an idea you had already filed. Status and Tags are gone with it; a Ticket has neither, and offering them is what created the fork in the first place. {% endcomment %}
